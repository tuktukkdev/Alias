# Материалы по тестированию — Alias Online

## 1. Цели и задачи тестирования

### Проверяемые аспекты

| Аспект | Цель |
|---|---|
| Функциональность | Убедиться, что все функции работают согласно требованиям |
| Нагрузка | Проверить стабильность под пиковым трафиком |
| Безопасность | Выявить уязвимости входных данных и авторизатции |

### Критерии приёмки

- Все юнит-тесты проходят без ошибок (`npm test`)
- Покрытие кода: ≥ 55 % по функциям для `authService`, ≥ 65 % по строкам для `wordService`
- При нагрузке 20 VU: p95 времени ответа ≤ 500 мс, доля ошибок ≤ 2 %
- Критические сценарии (вход/регистрация/создание комнаты) возвращают ожидаемые HTTP-статусы

## 6. Нагрузочные тесты

### 6.1 auth-load.js — Аутентификация

**Сценарий:** Регистрация нового пользователя → вход с его данными.

| Параметр | Значение |
|---|---|
| Инструмент | k6 |
| Этапы | 30 с разгон до 10 VU → 60 с стабильно → 30 с пик 25 VU → 30 с спад |
| Длительность | ~2.5 минуты |
| Thresholds | p95 < 500 мс, error_rate < 1% |

**Предусловие:** сервер должен быть запущен до старта теста. Если сервер недоступен, k6 завершится с ошибкой в фазе `setup` и сообщит об этом в консоли.

Запуск:
```bash
# Терминал 1 — запуск сервера
cd apps/server && npm run dev

# Терминал 2 — нагрузочный тест
k6 run apps/server/tests/load/auth-load.js
```

### 6.2 room-load.js — Игровые комнаты

**Сценарий:** Создание комнаты → получение состояния → подключение второго игрока → чат → выход.

| Параметр | Значение |
|---|---|
| Инструмент | k6 |
| Этапы | 20 с разгон 5 VU → 60 с стабильно 20 VU → 20 с спад |
| Длительность | ~1.7 минуты |
| Thresholds | p95 < 400 мс, error_rate < 2% |

**Предусловие:** сервер должен быть запущен до старта теста. Если сервер недоступен, k6 завершится с ошибкой в фазе `setup`.

Запуск:
```bash
# Терминал 1 — запуск сервера
cd apps/server && npm run dev

# Терминал 2 — нагрузочный тест
k6 run apps/server/tests/load/room-load.js
```

## 8. Найденные дефекты и их устранение

| ID | Дефект | Серьёзность | Причина | Исправление |
|---|---|---|---|---|
| BUG-01 | Хранимый хэш некорректного формата (без `:`) вызывал `timingSafeEqual` с разными длинами буферов | Средняя | Не проверялся формат хэша | Добавлена проверка `if (!salt \|\| !hash) return false` в `verifyPassword` |
| BUG-02 | `app.options("*", ...)` падал с ошибкой в Express 5 из-за path-to-regexp | Высокая | Несовместимость синтаксиса | Заменено на `app.options(/.*/, ...)` |
| BUG-03 | При пустом `wordPool` (`null`) вызов `filter` приводил к TypeError | Низкая | `wordPool` может быть `null` до `loadWordPool` | Добавлена проверка `if (pool && pool.length > 0)` |
| BUG-04 | Параметры маршрута `:roomId` могли иметь тип `string \| string[]` | Низкая | Типы Express v5 | Добавлена нормализация `String(req.params.roomId)` |
| BUG-05 | Нагрузочный тест `room-load.js` выполнял 480 901 итерацию с `error_rate=100%` при незапущенном сервере — все запросы мгновенно завершались ошибкой `connection refused`, счётчики раздувались | Высокая | Отсутствие предварительной проверки доступности сервера | Добавлена функция `setup()` в оба load-скрипта: если сервер недоступен, тест прерывается с понятным сообщением ещё до фазы нагрузки |

## 9. Установка зависимостей и запуск тестов

### 9.1 Установка зависимостей (Node.js / Jest)

Требуется Node.js 18+ и npm. Установка зависимостей сервера:

```bash
cd apps/server
npm install
```

Установленные тестовые пакеты (devDependencies):
- `jest` — тест-раннер
- `ts-jest` — поддержка TypeScript в Jest
- `@types/jest` — типы для Jest
- `supertest` — HTTP-интеграционное тестирование
- `@types/supertest` — типы для supertest

### 9.2 Установка k6 (нагрузочные тесты)

k6 — отдельный бинарный инструмент, не устанавливается через npm.

**Windows (через winget):**
```bash
winget install k6 --source winget
```

**Windows (через Chocolatey):**
```bash
choco install k6
```

**macOS:**
```bash
brew install k6
```

**Linux (Debian/Ubuntu):**
```bash
sudo gpg -k
sudo gpg --no-default-keyring --keyring /usr/share/keyrings/k6-archive-keyring.gpg \
  --keyserver hkp://keyserver.ubuntu.com:80 --recv-keys C5AD17C747E3415A3642D57D77C6C491D6AC1D69
echo "deb [signed-by=/usr/share/keyrings/k6-archive-keyring.gpg] https://dl.k6.io/deb stable main" \
  | sudo tee /etc/apt/sources.list.d/k6.list
sudo apt-get update && sudo apt-get install k6
```

Проверка установки:
```bash
k6 version
```

Подробнее: https://k6.io/docs/get-started/installation/

### 9.4 Запуск нагрузочных тестов

> **Важно:** перед запуском нагрузочных тестов сервер должен быть запущен.
> Если сервер недоступен, k6 завершится с ошибкой в фазе `setup` с понятным сообщением.

```bash
# Терминал 1 — запустить сервер
cd apps/server && npm run dev

# Терминал 2 — нагрузочный тест аутентификации
k6 run apps/server/tests/load/auth-load.js

# Терминал 2 — нагрузочный тест комнат
k6 run apps/server/tests/load/room-load.js

# Против удалённого сервера
k6 run -e BASE_URL=https://bluefieldapps.com apps/server/tests/load/room-load.js

# С сохранением результатов в JSON
k6 run --out json=results.json apps/server/tests/load/room-load.js
```
